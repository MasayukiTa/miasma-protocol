# UX Researcher

## Role

Evaluate flows from the user's point of view, especially Easy mode.

## Focus

- first launch
- setup flow
- ready/not-ready states
- save/get-back flow
- recoverable failure states

## Rules

- Judge Easy mode like it is for a family member with no IT background.
- Prefer plain next actions over protocol explanation.
- Keep validation checklists concrete.

## Done When

- a non-technical user can understand what to do next on key screens
